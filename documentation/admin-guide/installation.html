Installation Guide

Deploy SafeSphere in your environment

On this page

System Requirements

Minimum Requirements

Component Requirement
Operating System Linux (Ubuntu 20.04+), macOS, Windows 10/11
RAM 4GB minimum, 8GB recommended
Disk Space 2GB free space
Node.js Version 18.0.0 or higher
Python Version 3.9 or higher
PostgreSQL Version 13 or higher

Installation Methods

SafeSphere can be installed using two methods:

  • Docker Compose - Recommended for production deployments
  • Manual Installation - For development or custom deployments

Docker Installation

Recommended Docker installation is the easiest and fastest way to deploy SafeSphere.

Prerequisites

  • Docker Engine 20.10+
  • Docker Compose 2.0+

Installation Steps

1. Clone the repository:

git clone https://github.com/boniyeamincse/SafeSphere.git
cd SafeSphere

2. Configure environment variables:

# Create .env file
cp .env.example .env

# Edit .env with your settings
nano .env

3. Start the services:

# Build and start all services
docker-compose up -d

# View logs
docker-compose logs -f

4. Access the application:

  • Frontend: http://localhost:5173
  • Backend API: http://localhost:8000
  • API Docs: http://localhost:8000/docs

Manual Installation

Step 1: Install Dependencies

Install Node.js:

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# Verify installation
node --version
npm --version

Install Python:

# Ubuntu/Debian
sudo apt-get update
sudo apt-get install python3.9 python3.9-venv python3-pip

# Verify installation
python3 --version
pip3 --version

Install PostgreSQL:

# Ubuntu/Debian
sudo apt-get install postgresql postgresql-contrib

# Start PostgreSQL
sudo systemctl start postgresql
sudo systemctl enable postgresql

Step 2: Database Setup

# Create database and user
sudo -u postgres psql

CREATE DATABASE safesphere;
CREATE USER safesphere_user WITH PASSWORD 'your_secure_password';
GRANT ALL PRIVILEGES ON DATABASE safesphere TO safesphere_user;
\q

Step 3: Backend Setup

# Navigate to backend directory
cd backend

# Create virtual environment
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt

# Configure environment
cp .env.example .env
nano .env

# Run migrations (if using Alembic)
alembic upgrade head

# Start backend server
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000

Step 4: Frontend Setup

# Navigate to frontend directory
cd frontend

# Install dependencies
npm install

# Configure environment
cp .env.example .env
nano .env

# Start development server
npm run dev

Production Deployment

Important Never use development servers in production. Use proper web servers and process managers.

Backend Production Server

# Install gunicorn
pip install gunicorn

# Run with gunicorn
gunicorn app.main:app -w 4 -k uvicorn.workers.UvicornWorker -b 0.0.0.0:8000

Frontend Production Build

# Build for production
npm run build

# Output will be in dist/
# Serve with nginx or similar web server

Nginx Configuration

server {
    listen 80;
    server_name safesphere.yourcompany.com;

    # Frontend
    location / {
        root /var/www/safesphere/frontend/dist;
        index index.html;
        try_files $uri $uri/ /index.html;
    }

    # Backend API
    location /api {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }
}

Post-Installation

  1. Create an admin account
  2. Configure SMTP settings (see SMTP Setup)
  3. Set up user groups
  4. Configure system settings
  5. Test with a sample campaign
Installation Complete! Proceed to Configuration to set up your system.
